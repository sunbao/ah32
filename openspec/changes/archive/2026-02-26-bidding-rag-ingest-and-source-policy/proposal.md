## Why

招投标流程中的“合同库/案例库/模板库/政策法规库”等关键资料往往不在当前项目 RAG 中，导致对话式检索无法命中；用户需要系统明确反馈“库缺失/未导入”，并给出可执行的导入路径（上传/URL 抓取入库），从而把对外要提供的数据能力真正增强到招投标流程里。

同时，当前回答/写回默认展示来源（source/URL）会干扰正文排版；需要把“是否展示来源”变成由对话意图决定的能力，而非硬编码默认行为。

## What Changes

- 在招投标相关技能中，将“RAG 0 命中 = 缺库”作为可强制的业务约束：必须明确告知当前知识库暂无资料，并引导用户补库（上传/导入或提供 URL 抓取入库）；抓取失败时要求用户粘贴纯文本（暂不支持图片）。
- 架构层提供可复用的“缺库提示机制/能力开关”，由技能决定是否启用，避免把业务规则写死在 core。
- 新增“URL → 抓取正文 → 入库 → 可检索”的端到端能力，优先落地到当前招投标技能，并支持后续其他技能复用。
- **BREAKING**：默认不在正文/写回中展示 source/URL；仅当用户明确要求“出处/来源/引用/链接”等时才展示。
- 明确本阶段只覆盖《加快招标投标领域人工智能推广应用的实施意见》相关技能场景：检索不到时必须反馈缺口与下一步操作，不引入关键词检索，沿用向量检索。

## Capabilities

### New Capabilities

- `rag-missing-library-guidance`: 技能可声明并强制执行“RAG 0 命中缺库反馈 + 补库操作指引（上传/导入/URL 抓取；失败兜底为粘贴纯文本）”的对话行为契约。
- `rag-url-ingest`: 提供 URL 抓取正文并导入 RAG 的能力（基于现有浏览器控制层），默认进入当前项目库/项目索引，确保后续对话可向量检索命中。
- `source-display-policy`: 提供“来源展示策略”能力：默认不展示来源，按对话意图决定是否输出出处/链接，避免破坏正文与写回排版。

### Modified Capabilities

- (none)

## Impact

- Core/Agent：RAG 0 命中的提示机制与来源展示策略（提示词与路由/编排逻辑）。
- Skills：招投标相关技能（如 bidding-helper）的系统提示词、能力声明与工具编排（缺库反馈、URL 入库引导）。
- Integrations：复用现有 Playwright 浏览器控制层以抓取网页正文（失败需可诊断并提示用户兜底操作）。
- Backend APIs：复用/补齐 RAG 文本上传/导入接口与“项目库”归属/索引更新路径，保证入库后可检索。
