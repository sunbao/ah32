# 安全政策（阿蛤 / AH32）

## 报告漏洞

如果你发现潜在安全问题（例如：密钥泄露、鉴权绕过、任意文件访问等），请不要公开创建 issue。

请使用 GitHub 的 “Report a vulnerability / 安全漏洞报告”（Security → Advisories）提交私密报告。

如果你的 GitHub 账号/权限不方便使用该入口，请在 issue 中仅描述**不含敏感细节**的现象与复现步骤，并明确标注“安全问题”，同时避免粘贴任何密钥、Token、内网地址、用户数据或可利用 payload。

## 安全建议（使用者）

- 不要将 `.env` 提交到公开仓库或共享给第三方
- 生产环境请设置强随机 `AH32_API_KEY`/`VITE_API_KEY`
- 生产环境请收紧 CORS 配置，不要使用通配符
